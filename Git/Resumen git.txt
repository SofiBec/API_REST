Git

Repositorio remoto: https://github.com/SofiBec/TheFastFood.git

Comandos:

git init 			  	  		//para inicializar el repositorio
git config
git config --global user.name	//para elegir el nombre del usuario que comitea el repositorio
git config --global user.mail	//para elegir el mail del usuario que comitea el repositorio
git config --global --add safe.directory C:/xampp/htdocs //para hacerlo master
git status						//para conocer los estados de los archivos
git add							// para agregar el archivo o carpeta que vamos a rastrear, en este caso primer_proyecto/
git add	.						// para agregar todos los archivos o carpetas que esten en estado untraqued para ser preparados para el commit
git commit .m ""				//para guardar las versiones y poner un mensaje
git log							//para ver el historial
git checkout (hash del commit)	//se coloca el hash del commit al que quiero apuntar o ver en mi directorio
git checkout master				//para volver a posicionarse en la última versión, en mi caso usé:
git switch -f master			//por que mi archivo no se encuentra en el head
git remote add origin https...	//para agregar un directorio remoto y poder compartir el proyecto
git push -u origin master		//para subir todos los cambios al escritorio remoto, yo usé:
git push --set-upstream origin master


status de los archivos en git:
1) untracked: archivos sin rastrear o no trazado (rojo)
2) changes to be committed: archivo preparado o trazado, esperando para guardar los cambios (verde)

El HEAD es el que me indica la posición del último COMMIT, el que apunta o visualiza en nuestro directorio de trabajo.
Los commits tienen distitas identificaciones, se identifican por el hash.
